Highlight Options

This dialog allows you to set up rules for highlighting text in the logs.

When a line of a log file is displayed, its contents are checked against each of these rules. The first rule to match is applied. All rules below the matching rule are ignored.

The last rule is always empty, which means it always matches. Therefore, it can be used to define the standard style of the log.

Writing expressions

Each rule defines a regular expression that is used to determine if its styles should be applied to a log line.

For example, to match all lines containing the word Error, just enter the Error expression.

To match any line containing one or more A's, enter A+.

To match a whole line, you can use the usual ^ (start) and $ (end) anchors. So, to mach lines starting with [INFO], enter ^\[INFO\] (notice that the brackets must be escaped).

To make the expression case-insensitive, start it with (?i).

For assistance writing Java regular expressions, check the Oracle Regex Tutorial.

Choosing the style for a rule

If a rule matches, the background and text colors selected in the first and second color pickers, respectively, are applied to the log line.

A color may be specified by:

  • name: e.g. blue, yellow.
  • hexa-decimal components: e.g. 0x00ff00.
  • web value: e.g. #C0FFEE, #C0F.

See the Color class documentation for all options.
